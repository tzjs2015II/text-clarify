# 文本去重工具 - 项目总结

## 项目概述

成功开发了一个高效的大规模文本去重工具，专为处理几千到几万本TXT文件而设计。

## 核心技术

### 1. SimHash算法
- **64位指纹生成**: 快速生成文本的唯一标识
- **海明距离计算**: 高效判断文本相似度
- **智能分词**: 支持中文和英文文本处理

### 2. SQLite缓存系统
- **轻量级数据库**: 无需额外依赖，使用Python标准库
- **WAL模式**: 提升并发性能和数据安全性
- **断点续传**: 支持增量处理，避免重复计算

### 3. 智能去重逻辑
- **字数优先**: 相似文件中保留字数最多的版本
- **可调阈值**: 支持自定义相似度判断标准
- **批量处理**: 高效处理大规模文件集合

## 快速开始

### 命令行使用

```bash
# 试运行（推荐）
python text_deduplication.py ./books

# 真正删除
python text_deduplication.py ./books --execute

# 高级版本
python text_deduplication_advanced.py ./books --log-level DEBUG
```

### Python API使用

```python
from text_deduplication_advanced import TextDeduplicator

deduplicator = TextDeduplicator(
    root_dir="./books",
    db_path="cache.db",
    similarity_threshold=3
)

result = deduplicator.run_deduplication(
    recursive=True,
    dry_run=True
)

deduplicator.close()
```

## 相似度阈值

| 阈值 | 严格程度 | 说明 | 适用场景 |
|------|---------|------|---------|
| 1-2 | 非常严格 | 只删除几乎完全相同的文件 | 精确去重 |
| 3 | 平衡模式 | 删除内容相似的文件 | 通用场景（推荐） |
| 4-5 | 宽松模式 | 删除更多相似文件 | 激进去重 |

## 故障排除

### 常见问题

1. **内存不足**: 分批处理文件
2. **处理速度慢**: 降低日志级别，调整阈值
3. **相似度判断不准确**: 调整相似度阈值
4. **数据库锁定**: 确保只有一个实例在运行
